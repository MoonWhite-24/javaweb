# JavaWeb Market 2.0 项目部署指南

## 项目概述

JavaWeb Market 2.0 是一个基于 Spring Boot 3.x + Vue 3 的多模块电商平台，部署在 3 节点集群上，支持负载均衡、分布式 ID 生成、缓存加速和消息队列异步处理。

### 技术栈

| 层级 | 技术 | 版本 |
|------|------|------|
| **后端框架** | Spring Boot 3.x（嵌入式 Tomcat） | 3.x |
| **构建工具** | Apache Maven | 3.9.9 |
| **JDK** | OpenJDK (Red Hat Build) | 21.0.11 LTS |
| **数据库** | MariaDB (MySQL 8.0 兼容) | — |
| **缓存** | Redis (Lettuce 客户端) | 7.x |
| **消息队列** | Apache Kafka | 4.2.0 |
| **前端框架** | Vue 3 + Vite + Pinia + Element Plus | 5.4.x |
| **Node.js** | Node.js | 22.22.2 |
| **反向代理** | Nginx | 1.26.3 |
| **CI/CD** | Jenkins | 2.555.2 |
| **操作系统** | Rocky Linux | 10.1 |
| **SSH** | OpenSSH (密钥认证) | — |

### 核心功能模块

| 功能 | 实现方式 |
|------|----------|
| 用户认证 | JWT (access + refresh token)，BCrypt 密码加密 |
| 权限控制 | 自定义拦截器（`@RequireLogin` / `@RequireAdmin`） |
| 商品管理 | MyBatis 分页查询，多条件筛选，分类树 |
| 购物车 | Redis Hash 存储，支持商品数量修改和删除 |
| 订单系统 | 雪花算法订单号，超时自动取消（Kafka 延迟消息） |
| 秒杀活动 | Redis 预减库存 + Lua 脚本原子操作 + 分布式锁 |
| 接口限流 | 令牌桶算法 |
| 分布式 ID | Snowflake（Worker ID 由 systemd 环境变量注入） |
| 操作日志 | AOP 切面 + 注解（`@OperateLog`） |
| 健康检查 | Nginx upstream health check + 应用 `/api/health` 端点 |

## 一、应用配置文件

### 1.1 market2 systemd 服务

**所有 3 个节点都需要此文件，但 `SNOWFLAKE_WORKER_ID` 不同。**

**`/etc/systemd/system/market2.service`（Master 节点）：**

```ini
[Unit]
Description=JavaWeb Market 2.0
After=network.target

[Service]
User=root
WorkingDirectory=/opt/javaweb2.0
Environment="SNOWFLAKE_WORKER_ID=1"
ExecStart=/usr/bin/java -jar /opt/javaweb2.0/market-boot.jar
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

> **Node1 将 `Environment="SNOWFLAKE_WORKER_ID=2"`，Node2 改为 `3`。**
>
> **特别注意：** Environment 的引号必须闭合（`=1"` 不能写成 `=1`）。

### 1.3 Nginx 负载均衡配置

**`/etc/nginx/conf.d/market.conf`（Master 节点）：**

```nginx
# 后端集群（最少连接策略）
upstream market_cluster {
    least_conn;
    server 192.168.200.100:8080 weight=10 max_fails=3 fail_timeout=30s;
    server 192.168.200.101:8080 weight=10 max_fails=3 fail_timeout=30s;
    server 192.168.200.102:8080 weight=10 max_fails=3 fail_timeout=30s;
    keepalive 64;
}

# 限流区域
limit_req_zone $binary_remote_addr zone=api:10m rate=100r/s;
limit_req_zone $binary_remote_addr zone=seckill:10m rate=50r/s;
limit_conn_zone $binary_remote_addr zone=connperip:10m;

# 自定义日志格式（含 trace_id 和 upstream 信息）
log_format market_format '$remote_addr - $remote_user [$time_local] '
    '"$request" $status $body_bytes_sent '
    '"$http_referer" "$http_user_agent" '
    '$request_time $upstream_response_time '
    '$upstream_addr $http_x_trace_id';

server {
    listen 80;
    server_name _;
    charset utf-8;

    # 静态资源缓存（7 天）
    location /static/ {
        root /opt/tomcat/webapps/ROOT;
        expires 7d;
        add_header Cache-Control "public, immutable";
    }

    # 秒杀接口特殊限流
    location /seckill/do {
        limit_req zone=seckill burst=20 nodelay;
        limit_conn connperip 10;
        proxy_pass http://market_cluster;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header X-Trace-Id $http_x_trace_id;
        proxy_connect_timeout 5;
        proxy_read_timeout 30;
    }

    # 通用 API 代理
    location / {
        limit_req zone=api burst=100 nodelay;
        proxy_pass http://market_cluster;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header X-Trace-Id $http_x_trace_id;
        proxy_connect_timeout 30;
        proxy_read_timeout 60;
        proxy_send_timeout 30;
    }

    # Nginx 自身健康检查端点
    location /nginx-health {
        access_log off;
        return 200 "OK";
    }

    access_log /var/log/nginx/market-access.log market_format;
    error_log /var/log/nginx/market-error.log warn;
}
```

Master 节点必须能免密 SSH 到所有节点（包括自己），Jenkins 构建的部署和健康检查依赖此配置。

```bash
# 复制公钥到 3 个节点

ssh-copy-id root@192.168.200.101  # Node1
ssh-copy-id root@192.168.200.102  # Node2

# 验证
ssh root@192.168.200.101 "echo OK"
ssh root@192.168.200.102 "echo OK"
```

### 步骤 5：创建 market2 systemd 服务

**在每台节点上分别执行：**


```bash

systemctl disable market 2>/dev/null
systemctl mask market 2>/dev/null

# 确认旧 Tomcat 进程不存在
ps aux | grep catalina | grep -v grep
# （应无输出）
```

## 四、故障排查

### 4.1 服务 404 Not Found

**症状：** `curl http://IP:8080/api/health` 返回 404

**诊断：**

```bash
# 检查是谁在监听 8080
ss -tlnp | grep 8080

# 如果显示 Tomcat 而非 Spring Boot：
# java ... org.apache.catalina.startup.Bootstrap start
# 说明旧 Tomcat 占用了端口
```

**修复：**

```bash
# 杀掉旧 Tomcat
systemctl stop market 2>/dev/null
fuser -k 8080/tcp

# 重启 market2
systemctl restart market2

# 永久禁用旧服务
systemctl mask market
```

### 4.2 服务 403 Forbidden

Spring Security 拦截了请求。确认 JWT Filter 已通过白名单放行 `/api/health` 端点。

### 4.3 构建后前端页面不更新

前端 `dist/` 目录中的资源文件使用 Vite 的 hash 命名（如 `index-8uUV50R1.js`），旧文件不会自动清理。

```bash
# 清理 Node2 的旧 dist
ssh root@192.168.200.102 "rm -rf /opt/javaweb2.0/vue-market/dist/*"
# 然后重新部署
```

### 4.4 Snowflake ID 重复

如果两个节点的 Worker ID 相同，可能产生重复 ID。确认各节点 systemd 环境变量正确：

```bash
# 在每个节点执行
grep SNOWFLAKE /etc/systemd/system/market2.service
# Master 应显示: Environment="SNOWFLAKE_WORKER_ID=1"
# Node1  应显示: Environment="SNOWFLAKE_WORKER_ID=2"
# Node2  应显示: Environment="SNOWFLAKE_WORKER_ID=3"
```

### 4.5 Git push 失败（master → main 分支映射）

本地分支是 `master`，GitHub 默认分支是 `main`：

```bash
# 正确推送
git push origin master:main
```

### 4.6 VM 重启后恢复流程

```bash
# 1. 确认所有基础服务已启动
systemctl status market2 nginx jenkins mariadb redis 2>/dev/null

# 2. 如 Kafka 未自动启动
su - kafka -c "/usr/local/kafka/bin/kafka-server-start.sh -daemon /usr/local/kafka/config/server.properties"

# 3. 验证
curl http://localhost:8080/api/health
curl http://localhost:9090/login

# 4. 如果旧 Tomcat 又出现了（market.service 被 unmask）
systemctl stop market
kill $(ps aux | grep catalina | grep -v grep | awk '{print $2}') 2>/dev/null
systemctl restart market2
```

### 4.7 Maven 构建失败

```bash
# 检查依赖
cd /opt/javaweb2.0
mvn dependency:resolve

# 清理本地缓存
mvn clean
rm -rf ~/.m2/repository/com/market/

# 重新构建
mvn package -DskipTests
```
