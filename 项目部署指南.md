# JavaWeb Market 2.0 项目部署指南

## 项目概述

JavaWeb Market 2.0 是一个基于 Spring Boot 3.x + Vue 3 的多模块电商平台，部署在 3 节点集群上。

### 技术栈

| 层级 | 技术 | 版本 |
|------|------|------|
| **后端框架** | Spring Boot 3.x | — |
| **构建工具** | Maven | 3.9.9 |
| **JDK** | OpenJDK | 21.0.11 |
| **前端框架** | Vue 3 + Vite | 5.4.x |
| **Node.js** | Node | 22.22.2 |
| **数据库** | MySQL 8.0 | — |
| **缓存** | Redis 7.x | — |
| **消息队列** | Kafka 4.2.0 | — |
| **反向代理** | Nginx | 1.26.3 |
| **CI/CD** | Jenkins | 2.555.2 |
| **操作系统** | Rocky Linux | 10.1 |

### 模块结构

```
javaweb2.0/
├── pom.xml                    # 父 POM（多模块管理）
├── market-api/                # API 层（Controller）
├── market-service/            # 业务逻辑层
├── market-dal/                # 数据访问层（MyBatis Mapper）
├── market-common/             # 公共模块（DTO、工具类）
├── market-security/           # 安全模块（JWT、拦截器）
├── market-boot/               # Spring Boot 启动模块
├── market-task/               # 定时任务模块
├── market-monitor/            # 监控指标模块
├── sql/                       # 数据库初始化脚本
│   └── sql/
│       ├── init.sql           # 建表语句
│       └── init_data.sql      # 初始数据
├── vue-market/                # 前端项目（Vue 3 + Vite）
├── Jenkinsfile                # Jenkins Pipeline 定义
└── create-freestyle-job.groovy # Jenkins Job 创建脚本
```

## 初次部署步骤

### 1. 环境准备（所有 3 个节点）

```bash
# JDK 21
yum install -y java-21-openjdk java-21-openjdk-devel

# Maven 3.9（仅 Master 需要构建）
yum install -y maven

# Node.js 22（仅 Master 需要构建前端）
dnf module install -y nodejs:22

# Nginx（仅 Master）
yum install -y nginx
```

### 2. 克隆项目（Master 节点）

```bash
# 配置 GitHub SSH（如未配置）
ssh-keygen -t rsa -b 4096 -C "jenkins@market"
# 将 ~/.ssh/id_rsa.pub 添加到 GitHub Settings → SSH Keys

# 克隆项目
cd /opt
git clone git@github.com:MoonWhite-24/javaweb.git javaweb2.0
```

### 3. SSH 免密登录配置（Master → 所有节点）

```bash
# 生成密钥（如果还没有）
ssh-keygen -t rsa -b 4096 -f ~/.ssh/id_rsa -N ""

# 复制公钥到所有节点
ssh-copy-id root@192.168.200.100
ssh-copy-id root@192.168.200.101
ssh-copy-id root@192.168.200.102
```

### 4. 数据库初始化（Master 节点）

```bash
mysql -u root -p < /opt/javaweb2.0/sql/sql/init.sql
mysql -u root -p < /opt/javaweb2.0/sql/sql/init_data.sql
```

### 5. 创建 market2 systemd 服务（所有 3 个节点）

```bash
cat > /etc/systemd/system/market2.service << 'EOF'
[Unit]
Description=JavaWeb Market 2.0
After=network.target

[Service]
User=root
WorkingDirectory=/opt/javaweb2.0
Environment="SNOWFLAKE_WORKER_ID=1"
ExecStart=/usr/bin/java -jar /opt/javaweb2.0/market-boot.jar
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF

# 注意：每个节点的 SNOWFLAKE_WORKER_ID 不同
# Master → 1, Node1 → 2, Node2 → 3

systemctl daemon-reload
systemctl enable market2
```

### 6. 配置 Nginx（Master 节点）

Nginx 配置文件位于 `/etc/nginx/conf.d/market.conf`，使用 `least_conn` 策略负载均衡到 3 个后端节点。

```bash
systemctl enable nginx
systemctl start nginx
```

### 7. 部署 Jenkins（Master 节点）

详见 [Jenkins部署指南.md](./Jenkins部署指南.md)

### 8. 禁用旧版 Tomcat 服务

旧版 Tomcat 通过 `market.service` 启动，会占用 8080 端口，必须禁用：

```bash
# 在所有 3 个节点上执行
systemctl stop market 2>/dev/null
systemctl disable market 2>/dev/null
systemctl mask market 2>/dev/null
```

### 9. 首次构建部署

通过 Jenkins 触发构建，或在 Master 上手动执行：

```bash
# Git 拉取最新代码
cd /opt/javaweb2.0
git pull origin main

# 后端构建
mvn clean package -DskipTests -q

# 前端构建
cd /opt/javaweb2.0/vue-market
npm ci
npm run build

# 部署到 3 节点
JAR=/opt/javaweb2.0/market-boot/target/market-boot-2.0.0.jar
DIST=/opt/javaweb2.0/vue-market/dist/

# Master
cp ${JAR} /opt/javaweb2.0/market-boot.jar
systemctl restart market2

# Node1
scp ${JAR} root@192.168.200.101:/opt/javaweb2.0/market-boot.jar
scp -r ${DIST}/* root@192.168.200.101:/opt/javaweb2.0/vue-market/dist/
ssh root@192.168.200.101 "systemctl restart market2"

# Node2
scp ${JAR} root@192.168.200.102:/opt/javaweb2.0/market-boot.jar
scp -r ${DIST}/* root@192.168.200.102:/opt/javaweb2.0/vue-market/dist/
ssh root@192.168.200.102 "systemctl restart market2"
```

## 日常运维

### 服务管理

```bash
# market2 服务
systemctl start market2     # 启动
systemctl stop market2      # 停止
systemctl restart market2   # 重启
systemctl status market2    # 查看状态
journalctl -u market2 -f    # 查看日志

# Nginx 服务
systemctl reload nginx      # 重载配置
systemctl restart nginx     # 重启

# Jenkins 服务
systemctl start jenkins
systemctl stop jenkins
```

## 故障排查

### 服务无法访问（HTTP 404/503）

1. 检查 market2 是否在运行：`systemctl status market2`
2. 检查 8080 端口归属：`ss -tlnp | grep 8080`（应该是 market-boot.jar，不是 Tomcat）
3. 如果旧 Tomcat 占用了 8080：`systemctl stop market && kill $(lsof -t -i:8080)`，然后 `systemctl restart market2`

### 构建失败

1. 检查 GitHub 连通性：`ssh -T git@github.com`
2. 检查项目目录完整性：`ls /opt/javaweb2.0/`
3. 检查 Maven 依赖：`cd /opt/javaweb2.0 && mvn dependency:resolve`
4. 查看 Jenkins 构建日志：http://192.168.200.100:9090 → market2-deploy → 构建编号 → Console Output

### VM 重启后恢复

```bash
# 确认所有服务已启动
systemctl status market2 nginx jenkins mysql redis 2>/dev/null

# 如 Kafka 未自动启动
/usr/local/kafka/bin/kafka-server-start.sh -daemon /usr/local/kafka/config/server.properties

# 验证
curl http://localhost:8080/api/health
curl http://localhost:9090
```
